# vControl V2

Lokal-first Multi-Agent-Runtime. Ein Kernel, vier Accessoren: Server, WebUI, Desktop, CLI.
Der Agent ist BГјrger erster Klasse вҖ” er liest, editiert und deployed seinen eigenen Source.

---

## Prinzipien

1. **Python ist die lebendige Codebase.** Kein Build, kein Frozen-Code. `serve restart` ist der einzige Deployment-Schritt. Der Agent liest und schreibt den Source direkt.
2. **Wenige, groГҹe Module.** 12 Kern-Module mit klaren Interfaces. Jeder Ordner hat eine Hauptdatei (`module/module.py`) als Г¶ffentliche API. Soft-Limit: 600 Zeilen pro Datei.
3. **Async-first.** Der gesamte Kernel lГӨuft auf asyncio. Threads nur dort, wo native Bibliotheken es erzwingen.
4. **Constructor-Injection.** Dienste bekommen ihre AbhГӨngigkeiten via `__init__`. Kein `getattr`, kein Service-Locator. `typing.Protocol` fГјr Interfaces.
5. **Zwei Sprachen, eine Grenze.** Alles auГҹer der WebUI ist Python. Die WebUI ist Svelte (JavaScript, kein TypeScript). Kommunikation via HTTP/WS.
6. **Server-Maschine = Heimat des Agenten.** Nur wo der Server lГӨuft, lebt der Agent. Desktop und CLI sind passive Accessoren.
7. **Keine lokalen Sprachmodelle.** STT und TTS ausschlieГҹlich Гјber Online-APIs (OpenAI, Groq, Mistral, etc.). Kein PyTorch, kein faster-whisper, keine lokalen Engines.
8. **Internationalisierung von Tag 1.** Jeder sichtbare String lГӨuft Гјber das I18n-System. Englisch ist die Fallback-Locale.
9. **Ordentliches Logging und Error-Handling von Tag 1.** Strukturierte Logs, klare Error-Klassen, kein `print`, kein `except Exception: pass`.
10. **Nur der agentische Pfad.** In dieser App macht nur Sinn, was mit Tools arbeiten kann. Kein separater non-agentic Streaming-Pfad вҖ” ein Modell ohne Tool-Support ist nutzlos. Chat lГӨuft immer als Agentic-Loop.
11. **Zeitzonen von Tag 1.** Jeder persistierte Timestamp ist UTC mit explizitem Offset (RFC3339/ISO 8601). UI rendert in der konfigurierten Zeitzone des Benutzers. Kein implizites `datetime.now()`, keine lokale Zeit in der Persistenz.

## Kommunikationsschicht

- `POST /api/rpc` вҖ” Methoden-Dispatcher (JSON `{method, args, kwargs}` вҶ’ UIApi-Delegate вҶ’ Response)
- `/ws` вҖ” WebSocket-Push (Server subscribed auf internen Event-Bus, leitet an Clients weiter)
- Kein Auth (Single-User-Local)

---

## I18n / Lokalisierung

- **Backend**: `utils/` enthГӨlt den I18n-Backend-Service. LГӨdt Locale-Dateien, lГ¶st Keys auf, managed Fallbacks.
- **Frontend**: `webui/src/lib/i18n.js` ist die Runtime. `webui/src/i18n/` enthГӨlt JSON-Dateien pro Locale. `data-i18n`-Attribute im HTML-Markup fГјr statische Texte, `AppI18n.t(key, params)` fГјr dynamische.
- **Englisch ist immer Fallback**. Fehlende Keys in anderen Locales fallen auf `en` zurГјck.
- Jeder benutzersichtbare String lГӨuft von Anfang an durch das System. Keine hartkodierten Texte.

---

## Logging & Error-System

- **Strukturierte Logs**: `logging`-basiert, pro Modul ein Logger (`vcontrol.chat`, `vcontrol.tools`, ...). Log-Level via Settings oder CLI-Flag.
- **Error-Klassen**: Basis-Hierarchie in `utils/errors.py`. DomГӨnen-Extensions pro Modul (`chat/errors.py`, `providers/errors.py`, ...).
- **Provider-Errors**: Klassifikation in `providers/errors.py` вҖ” `retryable` vs `fatal`, `rate_limit` vs `auth_error` vs `model_not_found`. Einheitliche User-Messages.
- **Kein `except Exception: pass`**, kein `print()` fГјr Fehler. Kein stilles Schlucken.
- **Agent-Error-Kommunikation**: Fehler werden als System-Messages an die betroffene Session gepusht und via `onChatError` an die UI gemeldet.

## Deployment-Szenarien

### Headless-Server

```
Installiert: core/ + server/ + webui/ + cli/
Start:       python cli/main.py serve start
Agent:       Vollzugriff auf den Source
```

### Desktop вҶ’ Remote-Server

```
Server:  core/ + server/ + webui/ + cli/
Desktop: desktop/ (nur webview + requests)
Start:   python desktop/main.py --host 192.168.1.5 --port 8420
Agent:   Lebt auf dem Server, nicht auf dem Desktop
```

### All-in-One (Entwicklung)

```
Installiert: alles
Start:       python server/main.py & python desktop/main.py
Agent:       Vollzugriff auf alles
```
